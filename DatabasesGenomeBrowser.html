IGV

The genome browser, Integrative Genomics Viewer (IGV), be used through the web browser or downloaded through the Broad Institute. It is recommended it is downloaded as there are options that are not available through the web version. The remarkability of IGV is that it can expediently handle genomic-sized datasets, such as that from BAM or SAM files (which will be discussed later), by adjusting the resolution based on region browsing. For small regions, the resolution can be very high. For wide regions, the resolution becomes low. This design came to be because IGV was originally developed to meet the needs for large projects, such as the Cancer Genome Atlas and 1000 Genomes Project, to make data visualization easy for all. 

Some genome assemblies are already in present in IGV, but additional genomes can be downloaded to the browser through a server. The server option, which requires internet access, can be found under the top menu bar at Genomes > Load Genome From Server. Data from external databases, which will be displayed in the form of tracks, can be downloaded through File > Load From Server. Other options to load data externally through the internet include using URL links to specific databases. Alternatively, local data files, such as BED and WIG files, can also be uploaded to IGV and be viewed as tracks. Use the embedded IGV below!   




UCSC Genome Browser

The database and genome browser, the UCSC Genome Browser, is primarily a eukaryotic genome browser that consists of 97 eukaryotic genomes across 49 mammals. The UCSC Genome Browser, hosted by the University of California at Santa Cruz, displays data (with external links to NCBI and Ensembl) through tracks for information on mRNA, EST, RefSeq Genes, variations (SNPs), histone modifications, gene expression, etc. The rate of update for the UCSC Genome Browser will vary based on priorities, as primates and model organism genomes are given higher priority. 

Some tracks that can be found on the UCSC Genome Browser includes the UCSC Genes, Conservation, and PhastCons tracks. The UCSC Genes track is a gene prediction set based on compiling data from RefSeq, GenBank, and UniProt databases. The Conservation track is displayed on the human and mouse genomes and shows conservation of the genes in the region for up to 100 different species. The PhastCons track uses the PhastCons program that has a two-state phylogenetic HMM, with one probabilistic state for conserved regions and one probabilistic state for non-conserved regions, to display the predicted conservation for the genes in the selected region. The PhastCons program utilizes genome-wide alignments with limited species (human, mouse, dog, zebrafish, and fly) from BLASTz and can be downloaded as Multiple Alignment Format (MAF) files. These are just some examples as there are more tracks on the UCSC Genome Browser for data on variation, gene expression, ENCODE, etc. The UCSC Genome Browser allows modification of tracks in display through five modes: Hide, Dense, Full, Squish, and Pack. It is of interest to note that the use Full in track display should be done sparingly due to the large number of track items displayed, and that the Pack option is usually better-suited. In the UCSC Genome Browser, alternative versions of the genomic assembly for an organism can be used. The Current default assembly for the human genome is GRCh38/hg38 from December 2013, which is updated for the gene and mRNA tracks, but has some tracks that are not yet mapped to hg38 from hg19. 

Uniquely, UCSC Genome Browser has its own set of tools that utilizes its database. We have seen before in a previous chapter the NCBI’s alignment tool Splign. The UCSC Genome Browser has its own alignment tool that is analogous to Splign and is called the BLAST-like Alignment Tool (BLAT). However, as previously stated, it does not have a graphical output and can align anywhere on the genome without specifying a piece of genome DNA. Another tool is a data-mining program called the UCSC Table Browser. This tool can retrieve data from the UCSC Genome Browser database in a fashion similar to BioMart and biomaRt, but with a different set of parameters and syntax. Please visit this page for a guide on how to use the Table Browser. The Table Browser also allows uploading of files to use as a search parameter value, given that the information in the file is in the correct format. The strongest benefit to the Table Browser is its ability to transmit data directly to the Galaxy server, which can be used to perform comparative genomic analysis.   

Ensembl

Similar in nature to the UCSC Genome Browser, Ensembl is genomic database and browser hosted by the European Molecular Biology Laboratory's European Bioinformatics Institute (EBI) in the United Kingdom, is primarily used for sequenced genomes of chordates. The database includes non-chordate model organisms, but not prokaryotes. However, the Ensembl database has non-vertebrate genomes through five subportals: Bacteria, Protist, Fungi, Plants, and Metazoa. Archaeal genomes can be found inside the bacteria subportal called EnsemblBacteria. Ensembl is a MySQL relational database that is updated approximately every four months. Users can perform general searches based on genes or species-specific search. Click here for a tutorial on navigating the Ensembl database. Records in Ensembl use unique identifiers in the form of 11 digits with a heading based on the type of molecule and the species. For example, the heading ENSG is used for human genes and ENST is used for human transcripts. Ensembl was originally built for the human genome, which is why headings for other species will be different. For example, the heading for mouse genes is ENSMUG.   

A general search in Ensembl can show general information on chromosomes and the number of genes, further broken down as protein-coding or non-coding. Clicking a gene will gather data from another database that displays the gene’s information and specific regions in detail. On the left-hand tab within the selected genes will display options for a variety of information found for the gene, including transcripts, variants, protein information, paralogs/orthologs, and external references. Clicking the link to a transcript will display the transcript panel with similar options of deeper information on the transcript. Transcript-specific information can also include exonic and intronic information, as well as links to the cDNA and protein. Within each panel is an embedded genome browser with tracks from GENCODE or other databases. Furthermore, Ensembl allows customization options for users to customize the dropdown menu.   

Ensembl BioMart

BioMart is a data-mining tool that utilizes information found through the Ensembl Database by clicking the BioMart option on the top menu bar. BioMart allows users to download data from Ensembl based on certain criteria used in the search. First, select the Database to use. The primary database is Ensembl Genes, but other databases include those specific for mouse strains, variants, and regulatory data. The BioMart options can change based on the version release, in which case if certain options are desired then archived version of the database must first be accessed through Ensembl Archives and BioMart selected. However, it is important to note that although certain BioMart tools may be available in archived databases, those archived databases do not contain the same up-to-date information. 
   
Within BioMart are subcategories listed under two main categories: Filters and Attributes. The Filters category allow you to select for genes that has certain features in the subcategories. Expand the subcategories to find what subcategory you can search for: Region, Gene, Phenotype, Gene Ontology, Multi Species comparisons, Protein Domains and Family, and Variant. For example, the Region subcategory can be used to pick out the physical region of chromosome, locations, coordinates, etc. The Gene subcategory allows you to pick out IDs, features with external databases, transcript counts, etc. The Attributes category allow you to display what you want to see by column in the result. The selected subcategories under Attributes are the headings of the column in the result. The top item under the Attribute link is the first column and so on. There are three subcategories for the attributes: Gene, External, Protein Domains, and Family.   

In summary, the result are the data are queried for based on the Filter category, then displayed based on these columns with each line down based on the Attributes category.  This means that you can have 1 gene and 10 results because they’re all transcripts of that gene. Although each record is unique the result table may not look unique if you don’t have all the attributes showing. Within the resulting table on the webpage, the Count feature at the top, next to Results, will show the number of genes found based on the filter selections. The Results tab will only show a preview of 10 rows, regardless of the number of rows found. This can be adjusted by changing the dropdown menu from 10 to All, which will open a separate webpage with all the results. The results can also be exported. It is also important to know that FASTA sequences can be exported with the results by using the radio buttons on top under Attributes.   

Bioconductor and biomaRt in R

Although it is not explicitly a database or genome browser, Bioconductor and biomaRt is discussed here because they are data-mining tools that requires a database to be used with it. Bioconductor is an open-source programming tool in R language and environment that can be used to create customized pipelines to mine data from different sources. One benefit of Bioconductor is that it is faster on the genome-scale when compared to using genome browsers or maps. Within R is the biomaRt package, which is technically a Bioconductor package and are not considered synonymous between each other. Bioconductor is the broader tool suite, while biomaRt is the specific package within the suite. For information on how to set up biomaRt in R, refer to this guide after R is installed. It is recommended to install R Studio rather than the regular R installer for more options. To utilize biomaRt, the Ensembl database and the specific dataset (based on the organism) must be loaded first. For accessing the Ensembl database for Homo sapiens genes, the command to do this in R is: 
                                  
                                ensembl <- useDataset(dataset = "hsapiens_gene_ensembl", mart = ensembl)   

Once the database is loaded, use the getBM() command in R to retrieve information. The concept of searching the database is identical to that of Ensembl’s BioMart, which is a web-based version of the R-based Bioconductor package. The getBM() command has four arguments: attributes, filters, value, and mart. The arguments will utilize the equal (=) sign to indicate argument values. The attributes parameter is for the columns to be displayed. The filters parameter contain the subcategories to filter your query by, with the values of filtering based on the value parameter. The mart parameter is usually already established when selecting the database and dataset. An example of a full command can be seen as:   

getBM(attributes = c('affy_hg_u133_plus_2', 'entrezgene_id'), filters = 'affy_hg_u133_plus_2', values = c("202763_at","209310_s_at","207500_at"), mart = ensembl) 

Note that the R-language allows the use of concatenation commands, c(), within the parameter values, which will create a index-based vector to link multiple values to the parameter. The example command above will show two columns ('affy_hg_u133_plus_2' and 'entrezgene_id') after filtering the database for the subcategory ('affy_hg_u133_plus_2') that contains the values (‘202763_at’, ‘209310_s_at’, and ‘207500_at’) in the database (ensembl) which was declared as a variable in the first sample command for useDataset(). In creating variables, the results of the getBM() command can be stored within another variable to be used for other R-based commands, such as head(). For more information on R-language commands, please visit this page. It is also important to note that the R-language does not require spacing and delineates based on reserved characters. Aside from the getBM() command, there are several others that can be used for accessing the database, such as listAttributes(), listMart(), etc. For a more thorough walkthrough on how to utilize biomaRt commands, please visit this page for the documentation and this page for a video guide.    

Galaxy

Although Galaxy is also not explicitly a database or genome viewer, it is a tool that heavily relies upon data uploaded from other databases, particular the UCSC Genome Browser. Galaxy is a web-based interface that is used to analyze comparative genomic data retrieved from public databases. The goals of comparative genomics are three-fold. The first is to identify DNA sequences in a genome that are functional, as functions are generally selected by evolution to be maintained. The second is to determine a biological role of functional sequences by comparing the unknown sequences to the known sequences, which is the concept behind the phylogenetic relationships between orthologous and paralogous genes. The third goal is to compare the evolutionary history of sequences through analysis of genomic data.   

Galaxy, itself, is an open-source framework that utilizes command-line (CML) computation tools and database information to execute workflows. These workflows are designed to be robust and flexible, while maintaining the reproducibility that science requires. The user interface of the webpage is designed to allow users without extensive programming knowledge to utilize bioinformatic tools in the same manner. The Galaxy server is one of the best tools for starting bioinformaticians that do not have a background in computer science and will encompass an exercise in the next chapter! Galaxy has the ability to convert data of different formats into the formats that are accepted and used by Galaxy.   

The server itself is maintained, and contributed to, by groups of academics and individual users, who are collectively known as the Galaxy community. Galaxy can be used through the cloud server or it can be installed locally. The full potential of Galaxy is not able to be summed up in a single chapter, if not this entire tool, and will need to be learned through use by tutorials and walkthroughs.
